## 2. 交付范围

当前交付内容为一个基于 Next.js 15 的双语官网项目，主要用于品牌展示、软件下载、功能介绍、设备支持说明、文档入口与社区导流。

### 2.1 已包含模块

- 首页
- 功能页与功能子页
- 下载页
- 设备支持页
- 文档页
- 社区页与 Marketplace 页
- 博客页与博客详情页
- 定价、许可、隐私、条款等信息页
- 中英文双语路由
- 基础 SEO 元信息与站点地图

### 2.2 不包含内容

- 后台管理系统
- CMS 内容编辑后台
- 用户账号系统
- 在线支付流程
- 自动化运营面板

也就是说，这个项目当前是“静态内容驱动的网站前台”，维护方式以代码仓库内文件更新为主。

## 4. 项目结构与职责划分

### 4.1 核心目录

- src/app/：页面路由与页面级布局
- src/components/：页面模块和基础 UI 组件
- src/data/：结构化静态数据
- src/lib/：站点常量、工具函数、SEO 相关逻辑
- src/i18n/：国际化路由与消息加载逻辑
- messages/：多语言文案
- public/：静态资源

### 4.2 日常维护最常改的文件

- 站点名称、域名、外链配置：src/lib/constants.ts
- 英文文案：messages/en.json
- 中文文案：messages/zh.json
- 导航结构：src/data/navigation.ts
- 博客数据：src/data/blog-posts.ts
- FAQ 数据：src/data/faq.ts
- 设备数据：src/data/devices.json
- 站点地图：src/app/sitemap.ts

## 6. 品牌信息维护

站点的品牌基础信息集中在 src/lib/constants.ts。

可维护项包括：

- 站点名称
- 站点简称
- 站点域名
- 站点描述
- GitHub 仓库链接
- Discord 邀请链接
- 品牌数量与设备数量展示值

如果项目交付给新品牌，通常第一步就是先替换这里的品牌配置。

---

## 7. 多语言内容维护

### 7.1 文案存放位置

- 英文：messages/en.json
- 中文：messages/zh.json

### 7.2 维护原则

- 先维护英文主文案，再同步中文
- 键名保持一致，不要只改一个语言文件
- 文案调整尽量只改 value，不改 key
- 如果新增文案，需要同时补齐两个语言文件

### 7.3 常见问题

如果页面出现文案缺失、显示 key 名称或构建报错，优先检查：

- 两个语言文件是否都存在同名 key
- 页面引用的 namespace 是否正确
- JSON 结构是否有逗号或括号错误

## 8. 常见内容更新入口

### 8.1 导航菜单

导航数据位于 src/data/navigation.ts。

可维护内容：

- 顶部导航顺序
- 一级导航项
- 二级导航项
- 导航跳转链接

### 8.2 博客内容

博客数据位于 src/data/blog-posts.ts。

可维护内容：

- 文章标题
- 摘要
- 发布时间
- 分类
- 阅读时长
- slug

新增博客时需要注意：

- slug 必须唯一
- 中英文标题和摘要建议同步维护
- 发布时间格式保持一致

### 8.3 FAQ

FAQ 数据位于 src/data/faq.ts。

适合维护：

- 常见问题标题
- 解答内容
- 问题顺序

### 8.4 设备支持列表

设备数据位于 src/data/devices.json。

适合维护：

- 设备名称
- 品牌名称
- 类型分类

维护建议：

- 保持品牌命名统一
- 保持设备类型字段统一，避免同义不同写
- 大批量更新前先备份原始数据

### 8.5 下载页内容

下载页的展示内容来自页面文案与站点常量。

重点检查：

- GitHub Releases 链接是否正确
- 平台描述是否匹配真实发布情况
- 版本说明是否需要同步更新

如果下载入口变更，优先检查 src/lib/constants.ts 中的 GitHub 仓库地址与下载页相关文案。

---

## 9. 静态资源维护

图片和静态资源位于 public/ 目录。

建议维护规范：

- 新资源按业务模块分类放置
- 文件命名使用英文和短横线
- 上线前压缩图片体积
- 确认素材版权可商用

如果替换首页主视觉、功能页图片、社区展示图，优先修改 public/images/ 下对应目录资源。

## 10. SEO 与站点基础配置

### 10.1 站点信息来源

站点基础信息主要来自：

- src/lib/constants.ts
- src/app/[locale]/layout.tsx

### 10.2 可维护项

- 站点名称与描述
- canonical 域名
- Open Graph 基础信息
- sitemap 页面覆盖范围
- robots.txt

### 10.3 上线前必查

- 域名是否为正式域名
- metadataBase 是否正确
- sitemap 链接是否全部可访问
- robots.txt 是否符合生产要求

## 11. 环境变量说明

当前项目依赖以下公开环境变量：

- NEXT_PUBLIC_SITE_NAME
- NEXT_PUBLIC_SITE_SHORT_NAME
- NEXT_PUBLIC_SITE_URL
- NEXT_PUBLIC_SITE_DESCRIPTION
- NEXT_PUBLIC_GITHUB_REPO
- NEXT_PUBLIC_DISCORD_INVITE
- NEXT_PUBLIC_BRAND_COUNT
- NEXT_PUBLIC_DEVICE_COUNT

这些变量通常用于品牌展示、SEO、下载入口和社区外链。

如果部署环境没有配置这些变量，项目会回退到代码中的默认值。

```bash
npm install
npm run dev
```

```bash

npm run start
```

## 13. 上线检查清单

正式上线前建议逐项确认：

- 品牌名、域名、Logo 已替换完成
- GitHub、Discord、下载链接已替换为正式地址
- 中英文页面均可访问
- 404 页面可正常显示
- sitemap 已生成
- robots.txt 可访问
- 隐私政策、条款、许可页面已由业务或法务确认
- 所有图片已确认版权
- 首页首屏、下载页、文档页在移动端正常显示
- npm run build 可通过

## 14. 日常维护建议

### 14.1 建议维护频率

- 每周检查下载链接是否有效
- 每周检查社区外链是否有效
- 每月补充博客或产品动态内容
- 每月校验设备列表与品牌数量展示
- 每次发版后同步更新下载页与相关文案

### 14.2 推荐维护流程

1. 在本地或预发环境修改内容
2. 自查中英文是否同步
3. 执行构建检查
4. 在预览环境确认页面效果
5. 再部署到正式环境

---

## 15. 常见问题排查

### 15.1 页面文案没有更新

优先检查：

- 是否改的是正确语言文件
- 是否部署到了正确环境
- 是否命中了旧缓存

### 15.2 某个页面打不开

优先检查：

- 路由文件是否存在
- 导航链接是否写错
- 多语言路径是否正确

### 15.3 构建失败

优先检查：

- JSON 文案文件格式
- 新增页面是否导出正确
- 引入路径是否错误
- 环境变量是否缺失

### 15.4 下载按钮跳转错误

优先检查：

- src/lib/constants.ts 中仓库地址是否正确
- GitHub Releases 页面是否存在
- 文案是否仍引用旧品牌链接
